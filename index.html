🎭 Assign Role & Access Level

Select the user's role. Each role has a predefined access scope enforced by Azure Row-Level Security — users can only see data relevant to their role.

👤
Caseworker
Access to their own assigned clients only. Can enter notes, log appointments, update housing status.
📊
Program Manager
Access to all clients in the org. Can run reports, view outcomes, manage caseworker assignments.
🛡️
Org Admin
Full org access. Can manage users, view audit logs, configure compliance settings and grant reports.
🔐 RLS Policy: Role assignment writes to Azure AD and automatically updates SESSION_CONTEXT(OrgID) and SESSION_CONTEXT(Role) on every database connection. The fn_OrgFilter predicate enforces this at the engine level — no application-level bypass is possible.
